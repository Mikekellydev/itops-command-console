Workflow

Built for operators who want speed and control.

Create

`itnew` validates input, allocates the next record key, creates the ticket folder, and inserts the DB row.

Manage

`itdash` keeps the active queue visible with keyboard-driven filtering, worklog entry, and state transitions.

Operate

`itops_ent` opens a tmux workspace with the dashboard, today’s log, and a shell for live operational work.

Install

One-command setup on Ubuntu or WSL.

The installer creates the database, Python environment, and local wrapper commands.

git clone https://github.com/mikekellydev/itops-command-console.git
cd itops-command-console
./install.sh

itops_ent

What’s Inside

Small surface area, clear boundaries.

Storage

SQLite stores records, worklog history, counters, and future time entries.

UI

Textual provides a responsive terminal dashboard with search and queue actions.

Artifacts

Markdown files keep longer-form ticket and daily log notes readable outside the app.

Git Hygiene

The public repo tracks code, templates, and examples, not live tickets, databases, or local virtualenvs.
