Use Cases

  • SysAdmin on call tracking incidents, decisions, and follow ups
  • MSP operations lightweight client record tracking without SaaS overhead
  • Homelab operator maintaining change history and troubleshooting notes
  • IT leadership demonstrating operational discipline and workflow design in interviews

Core Capabilities

  • Record types: INC, SR, CHG, PRB, KB
  • SQLite backed queue with structured worklog entries
  • Auto increment record keys per type via counters table
  • tmux command console layout: dashboard plus daily log plus shell
  • Fast hotkeys for hold, worklog, resolve, close, and filtering

Install on Your Computer

Target platform Ubuntu LTS or WSL Ubuntu. Requires tmux, sqlite3, and Python 3.

1. Install dependencies

sudo apt update
sudo apt install -y git tmux sqlite3 python3 python3-venv

2. Clone the repository

git clone https://github.com/mikekellydev/itops-command-console.git
cd itops-command-console

3. Create the Python environment for the dashboard

cd 95_Tools/dashboard
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install textual rich
deactivate
cd ../../

4. Create the ITOps directories

mkdir -p ~/ITOps/10_Tickets ~/ITOps/20_Areas/Daily ~/ITOps/80_Time/DB

5. Initialize the SQLite database

sqlite3 ~/ITOps/80_Time/DB/itops_enterprise.db ".tables"

If this is your first run, follow the schema in the repository docs to create tables.

6. Install CLI commands

mkdir -p ~/.local/bin
# Copy scripts from repo into ~/.local/bin if you keep them versioned in the repo
# Example:
# cp scripts/itnew ~/.local/bin/itnew
# cp scripts/itops_ent ~/.local/bin/itops_ent
# cp scripts/itdash ~/.local/bin/itdash
chmod +x ~/.local/bin/itnew ~/.local/bin/itops_ent ~/.local/bin/itdash

7. Run

itops_ent
